A bit different C++ tutorial

Introduction

Why should anyone want to read another C++ learning material when there are so many of them out there, both printed and online? Because, in my opinion, they generally do not provide a true understanding of C++. For example, how can anyone understand C++ without learning about the as-if rule and the observable behavior concept?

Typical C++ learning texts teach us how to use all those abstraction mechanisms such as functions, classes, or polymorphism, they teach us how to write data to a file or how to sort an array. But these mechanisms and functionalities are provided by many other programming languages as well. So why should we actually use C++ and what is so unique about it?

Of course, the highest motivation for using C++ is performance and efficiency. Performance in terms of fast programs and efficiency in terms of minimizing required system resources, such as CPU runtime or the amount of allocated memory. Without the need for performance and efficiency, one would likely not use C++ since it is not an easy language.

In fact, C++ is hard. Forget about learning it in 21 days. Yes, you can learn how to write a program with some desired functionality (which you would likely learn much easier with Python). But to learn how to write code with respect to maximizing performance and efficiency usually requires much more knowledge and insight.

For illustration, let us look at a question frequently asked C++ students or job applicants: “Where are function local variables stored?” A typical meaning of this question actually is: “Where are function local variables with automatic storage duration stored when the function is executed at runtime?” A common answer is: “On the stack.” We were tauhgt so. Even the books tell us that:

In a typical implementation, automatic objects are allocated on the stack…​

— Bjarne Stroustrup
The C++ Programmin Language (4th edition)

When a function is called, its automatic variables are added to the stack…​

— Stephen Prata
C++ Primer Plus (6th edition)

But this answer is incorrect. Moreover, the question itself is wrong since it does not make any sense. To understand, please, follow to the very first post of this tutorial…​

Note that this tutorial does not explain C++ completely from scratch. If you are a beginner with C++, or even with programming, I would recommend — simultaneosly with this tutorial — using some of the multitude of the “conventional” C++ learning sources, such as the books I quoted above.
